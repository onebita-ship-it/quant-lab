## 1. 설계

### 유니버스 (`config/universe.txt`, 재량 관리)
사람이 분기마다 수정하는 텍스트 파일. 티커별 `core/satellite` + `원지수` 표시.
```
TQQQ   core        QQQ
UPRO   core        SPY
SOXL   satellite   SOXX
```
> **새 티커를 넣어도 그 원지수 추세신호가 ON 되기 전엔 매수되지 않는다**(엔진 A가 신호 통과 자산만
> 후보로 삼음). 분기 리뷰 때 이 파일만 고치면 전략이 따라오는 구조.

### 엔진 A — 추세+모멘텀 로테이션
- 유니버스 중 **각자 원지수 200일선(기울기+스트릭5)이 ON**인 자산만 후보.
- 그중 **3·6개월 블렌드 모멘텀 1위**를 월말 리밸런스로 보유. ON 자산 없으면 SGOV.

### 엔진 B — 코어 폭락 계단투입
- **코어**(TQQQ·UPRO)의 원지수(QQQ·SPY)가 **최근1년 고점 대비 -30/-40/-50%** 도달 시,
  SGOV 대기현금을 **3등분 계단 투입**(각 티어 **QQQ 절반 + TQQQ 절반**).
- 어느 자산이든 **신호 재점등 시 엔진 B 청산 → 엔진 A 복귀.**

## 4. 몬테카를로 (5년 × 2000경로, joint block bootstrap)

멀티자산 상관 보존을 위해 QQQ/SPY/SOXX를 **같은 날짜 블록으로 동시 샘플링**.

| 지표 | V9 2엔진 통합 | V9 엔진A 단독 |
|---|---:|---:|
| CAGR 중앙값 | **3.5%** | **3.2%** |
| 총수익 중앙값 | +19.1% | +17.0% |
| 총수익 5%ile | -83.0% | -84.1% |
| 총수익 95%ile | +1271.6% | +1241.6% |
| MDD 중앙값 | -74.0% | -73.9% |
| MDD 5%ile(악화) | -91.2% | -91.6% |
| 손실확률 | 44.2% | 45.4% |
| 반토막확률 | **24.6%** | 25.7% |

**해석**
- **중앙 CAGR 3.5%, 반토막 확률 25%** — 백테스트 27%의 화려함은 *과거의 추세·모멘텀 지속성*에 기댄 값.
  블록 부트스트랩이 추세를 섞으면 무너진다(추세추종에 불리한 **비관적 하한**).
- **MC에선 통합 ≈ A단독**(3.5 vs 3.2%) — 부트스트랩은 다년 지속 하락장을 못 만들어 **엔진 B가 실제로
  피 흘리는 국면(닷컴·2022 같은 다년 베어)을 재현하지 못한다**. 즉 백테스트에서 드러난 엔진 B의
  -3.3%p·MDD 악화는 MC가 과소평가한다 → **실제 엔진 B 위험은 이 표보다 크다.**
- v8 S2(중앙 0.8%)보다 v9가 나아 보이는 건 멀티자산이 트렌딩 자산을 잡을 확률을 높여서지만, 여전히
  **20%는 미래 기대값이 아니다.**

## 5. 결론 & 추천

1. **v9에서 하나만 고른다면 → 엔진 A 단독**(엔진 B 미탑재). CAGR 27.1%·MDD -86%·샤프 0.70으로
   통합(23.8%/-97%)을 **전 지표에서 지배**. "2엔진 통합"이라는 설계 의도는 **백테스트로 반증**됨.
2. **엔진 B(폭락 물타기)는 통합하지 말 것.** 그 아이디어는 **v8 방식(외부 실탄·QQQ 중심·시가평가 분리)**
   으로만 살린다. 레버리지 자산을 포트폴리오 현금으로 falling-knife 매수하면 -97% MDD.
3. **유니버스 재량이 최대 리스크.** SOXL 하나로 CAGR이 7.7%p 흔들린다. 분기 유니버스 수정은
   **성과 드라이버이자 데이터마이닝 함정** — 규칙(예: 시총·유동성 기준)으로 재량을 제약할 것.
4. **목표(연 20%+) 관점**: 과거구간에선 엔진 A 단독이 27%로 최선이나, **MC 중앙 3%·MDD -86%**.
   결국 v8·v9 통틀어 **어떤 레버리지 추세/로테이션도 미래기대 20%를 견고히 넘기지 못한다**. 20%+는
   좋은 국면의 상단이다. 이를 추구하려면 **엔진 A 단독을 -80%+ MDD 감내 가능한 소액 사이징**으로,
   폭락 추가투입은 **별도 외부자금(v8 §6)** 으로 병행하는 조합이 가장 정직한 실전안.

## 6. 한계
- **합성 데이터**: 2010 이전 3x·SGOV, 2001 이전 SOXL은 합성 → 닷컴·GFC 충실도 원리상 검증 불가.
- **MC의 구조적 편향**: 블록 부트스트랩은 (a)추세추종을 과소평가하고 (b)다년 베어를 못 만들어 **엔진 B의
  해악도 과소평가** → MC는 상·하방 모두 왜곡. 백테스트 스트레스구간과 함께 봐야 한다.
- **유니버스 생존편향**: SOXL을 "지금" 넣은 건 반도체가 잘됐음을 아는 상태의 선택. 2005년 투자자가
  같은 유니버스를 짰을지는 알 수 없다.
- **세금·환전 미반영**(세전 비교). 로테이션·재점등 매매가 잦아 **세후 드래그가 v8보다 클 수 있다**(`result_tax.md`).
- **⑤ ON연환산**은 전 전략에 '유니버스 신호 하나라도 ON' 마스크를 공통 적용(상호 비교용) — v8 리포트의
  QQQ 단일 마스크 값과는 다르다.
- 3배 레버리지 꼬리위험·과거분포 유지 가정은 `result_final.md` 참조.
