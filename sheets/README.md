# sheets — 구글 시트 일일 기록·판단 시스템

확정 룰북(`../result_final.md §0`)을 **구글 시트 + Apps Script**로 옮긴 실전 운용 도구.
매일 아침 시트가 스스로 시세를 받아 **판단(오늘 할 일)을 내리고 하루 1행씩 기록**하며,
체결을 입력하면 규칙 준수·슬리피지를 자동 대조한다. 로컬 파이썬 없이 **폰에서도** 돌아간다.

> ⚠️ 이 도구는 기록·점검용이며 **자동매매가 아니다**. 주문·체결은 본인이 수행한다.
> 룰북 §0은 방화벽 — 이 시트는 룰을 **바꾸지 않고 그대로 코드화**했을 뿐이다(아래 패리티 감사).

## 구성

| 파일 | 역할 |
|---|---|
| `signals.gs` | 룰북 §0 계산 로직 (순수 함수) — `journal/common.py` 를 1:1 로 옮김 |
| `main.gs` | 시트 UI·자동화 (탭 생성, Yahoo 시세, 대시보드, 일일기록, 매매검증, 월간요약, 트리거, 메일) |
| `parity/` | **왜곡 감사 하네스** — JS 로직이 파이썬 원본과 일치하는지 전 구간 비교 |

만들어지는 탭:

| 탭 | 역할 | 대응 파이썬 도구 |
|---|---|---|
| `⚙️설정` | 파라미터 + 유니버스 표 | `journal/config.json` + `config/universe.txt` |
| `📊오늘` | 아침 브리핑 (게이트·오늘 할 일·파킹·위성) | `journal/daily_check.py` |
| `📅일일기록` | **하루 1행 자동 누적** (판단 근거 스냅샷) | (신규 — 시트 전용) |
| `✍️매매기록` | 체결 수기 입력 → 준수·슬리피지 자동 대조 | `journal/log_trade.py` |
| `📈월간요약` | 준수율·추적오차·성과 | `journal/monthly_report.py` |
| `_시세` | 시세 캐시 (숨김, Yahoo 실패 시 폴백) | `data/` |

## 설치 (최초 1회, 약 10분)

1. **새 구글 시트** 생성 (sheets.new) — 이름 예: `퀀트랩 매매일지`.
2. 메뉴 **확장 프로그램 → Apps Script** 열기.
3. 기본 `Code.gs` 내용을 지우고 → 이 저장소 `sheets/signals.gs` 내용 붙여넣기.
   좌측 **파일 + 버튼**으로 `main` 스크립트 파일을 추가하고 `sheets/main.gs` 내용 붙여넣기.
4. **프로젝트 설정(⚙️) → 시간대**를 `(GMT+09:00) Seoul` 로 변경.
5. 에디터 상단에서 함수 `setup` 선택 → **실행** → 권한 승인
   (스프레드시트·외부 URL 연결·메일 발송 — 본인 계정에서만 도는 스크립트다).
6. 시트로 돌아와 `⚙️설정` 탭에서 **`total_capital` 을 본인 코어 몫으로 수정**
   — 계좌 전체가 아니라 **코어 슬리브(42.5%)만!** (`journal/day1_checklist.md` 참조)
7. 새로고침하면 메뉴에 **[퀀트랩]** 이 생긴다 → **📊 오늘 브리핑 갱신**으로 첫 실행 확인.
8. **[퀀트랩 → ⏰ 매일 자동실행 설치]** — 매일 아침(기본 7시 KST) 자동으로
   시세 갱신 → 일일기록 1행 추가 → 대시보드 갱신 → (설정 시) 이메일 브리핑.

> 자동실행 시각은 `⚙️설정`의 `trigger_hour` (권장 6~9시 KST — 미국 장 마감 후).
> **시각을 바꾼 뒤에는 [⏰ 매일 자동실행 설치]를 다시 실행해야 적용된다** (트리거는 설치 시점 값으로 고정).
> `email_to` 에 이메일을 넣으면 **새 거래일 봉이 생긴 아침에만** 브리핑 메일이 온다
> — 미국 거래일 다음날 아침이므로 보통 **화~토요일(KST)** 에 오고, 일·월요일 아침은 무음.

## 매일 워크플로

1. **아침**: 자동 실행됨 — `📊오늘` 탭(또는 메일)에서 **[오늘 할 일]** 확인. 시키는 대로만.
2. **체결 후**: `✍️매매기록`에 한 줄 입력 → 메뉴 **[✍️ 매매기록 검증]** 실행.
3. **금요일(KST) 아침**: SGOV 매도 알림이 뜬다 → **그날 밤 미국 금요일 장**에서 다음 주
   5회분 예수금 확보 (룰북 ⑦). *금요일 알림은 실행일(KST) 기준이라 행동 가능한 시점에 온다.*
4. **월말**: 위성 리밸런스(브리핑이 알려줌) + **[📈 월간요약 갱신]**.

### ✍️매매기록 입력 방법

| 열 | 값 |
|---|---|
| 날짜 | `2026-07-08` 형식 |
| 슬리브 | `코어` / `위성` / `금` / `파킹` (드롭다운) |
| 구분 | `buy` `take_profit` `quarter` `deploy_reserve` (코어) · `buy`/`sell` (위성·금·파킹) |
| 티커 · 수량 · 체결가 | 체결 그대로 (수수료$는 비우면 가정치 0.07%로 계산) |

- **새 사이클 첫 매수는 플래그가 필요 없다** — 사이클이 비활성인 상태의 코어 `buy` 를
  자동으로 새 사이클 시작으로 인식한다 (1회분 = 그 시점 운용현금 ÷ 40 고정).
- 회색 열(기준종가·슬리피지bp·준수·체크상세…)은 검증 메뉴가 채운다. 손대지 말 것.
- 시트에는 숨은 상태 파일이 없다 — **매매기록 = 유일한 진실**. 코어 상태(회차·평단·현금)는
  매번 기록 전체를 리플레이해서 계산하므로, 과거 행을 고치면 상태도 그에 맞게 다시 계산된다.

## 판단 로직 ↔ 룰북 §0 매핑

| 룰북 | 시트 구현 |
|---|---|
| ① 진입 게이트 | QQQ 200일선 3조건 **AND** 13612W 카나리아(SPY·EFA·EEM·AGG, 월말 판정) |
| ② 분할 매수 | 1회분 = 사이클 시작일 운용현금 ÷ 40 고정, 매 거래일 1회분 (금액 ±5% 대조) |
| ③ 익절 +15% | 평가액(매도비용 반영) ≥ 투입원가 ×1.15 → 전량 매도 지시 |
| ④ 쿼터손절 | 40회분 소진 시 4일 ×1/4 청산 지시 (수량 ±5% 대조) |
| ⑤ 리저브 | `deploy_frac` 0.67 설정 시 고점대비 -30/-50% 발동 검사 (고점은 체결 기준종가로 추적) |
| ⑦ SGOV 파킹 | 미투입 현금 SGOV 유지 안내 + **금요일(KST) 아침** 다음 주 5회분 매도 알림 → 그날 밤 미국 장에서 실행 |
| ⑧ 위성 엔진A | 원지수 추세 ON 자산 중 3·6모멘텀 1위 타깃, 월말 리밸런스·OFF 즉시 교체 알림, 보유-타깃 불일치 경고 |

## 파이썬 `journal/` 과의 관계

같은 룰북의 두 가지 프런트엔드다. 병행해도 되고 시트만 써도 된다.

의도된 차이 4가지 (전부 시트가 §0 또는 실기록에 **더** 충실한 방향):
1. **신규 진입 준수 검사**: `log_trade.py` 는 추세만 검사하지만, 시트는 §0 ① 그대로
   **추세 AND 카나리아**를 검사한다.
2. **`deploy_reserve` 전이**: CLI는 발동 조건 미충족이면 이체 자체를 거부하지만, 시트는
   "기록 = 실제로 한 일"이므로 **기록대로 이체를 반영하되 위반(❌)으로 표시**한다.
3. **코어의 임의 `sell`**: CLI에는 아예 없는 입력이지만, 시트에서 기입하면 **상태에는
   반영하되 '룰북정의행동' 위반(❌)으로 표시**한다 (무시하면 유령 보유가 생기므로).
4. **금요일 SGOV 알림 시점**: `daily_check.py` 는 마지막 봉의 요일(=토요일 아침에 알림)로
   판정하지만, 시트는 **실행일(KST) 기준**이라 금요일 아침에 알림 → 그날 밤 실행 가능.

## 패리티 감사 (왜곡 검증)

JS 재구현이 파이썬 원본과 다르게 동작하면 그게 곧 룰 왜곡이다. 감사 하네스가
추세·카나리아·위성·달력 판정 전 구간, 실제 `log_trade.py` CLI 실행 vs JS 리플레이(리저브
발동/미발동 포함), 실제 `daily_check.py` CLI 출력 vs JS 처방(익절/쿼터/매수/대기 + 금액·파킹)을 비교한다:

```bash
pip install -r requirements.txt   # pandas·numpy
python sheets/parity/run_parity.py
```

`data/` 에 실데이터가 있으면 실데이터를 쓰고, 없는 티커는 **임시 디렉터리에** 합성 생성한다
(수식 비교라 유효). **운영 캐시 `data/` 에는 아무것도 쓰지 않는다** — 합성 시세가 실전 판정에
섞이면 안 되기 때문. **`signals.gs` 를 한 줄이라도 고치면 이 감사를 다시 통과시켜야 한다.**

최근 결과 (2026-07-08, 합성 7년·게이트 ON/OFF 균형 커버리지, 적대적 감사 후 재실행):

```
✅ 1. 추세 필터 전 구간 (표본 1750)      ✅ 5. 달력(월말·요일) 전 구간 (표본 1750)
✅ 2. latest_signal 격자 (표본 174)      ✅ 6. 리플레이 상태 + 행별 체크 (표본 10)
✅ 3. 카나리아 월별 (표본 78)            ✅ 7. 리저브 발동/미발동 (표본 32)
✅ 4. 위성 엔진A 격자 (표본 97)          ✅ 8. 오늘 할 일(daily_check) + 파킹 (표본 7)
→ 패리티 감사 통과 — 시트(JS) 로직이 파이썬 원본과 일치
```

## 한계·주의

- 시세는 Yahoo Finance 비공식 API(`query1.finance.yahoo.com`, 조정종가 = yfinance `auto_adjust`
  와 동일 기준)를 쓴다. 실패 시 `_시세` 캐시로 폴백하고 `📊오늘` 상단에 경고를 띄운다.
- 트리거를 미국 장중(밤 10:30~새벽 5시 KST)에 걸면 미완성 봉으로 판정하니 **아침 6~9시** 권장.
- 판정은 QQQ(기초지수), 매매는 TQQQ — 백테스트와 동일. 월말 판정은 영업일 근사(미국 휴장일 무시).
- 증권사 자격증명·API 키는 시트에도 스크립트에도 넣지 않는다 (`day1_checklist.md §4`).
